# Sales Letter Writer — Cattura la Realtà

## Panoramica

Questa skill produce copy di vendita per le pagine dei funnel. A differenza delle email
e dei post social, qui l'obiettivo è UNO: far cliccare il bottone di acquisto.
Ogni parola è calibrata sulla psicologia del cliente target e sulla roadmap delle convinzioni.

---

## Come usare questa skill

### Input richiesto

L'utente fornisce:
- **Tipo di pagina** (landing, OTO1, OTO2, downsell, checkout)
- **Prodotto** (nome, cosa include, prezzo, per chi è)
- **Target** (quale segmento/avatar)
- **Posizione nel funnel** (è la prima pagina? viene dopo un acquisto?)

### Output prodotto

- Headline principale + 2 varianti A/B
- Copy completo della pagina, sezione per sezione
- Testo del bottone CTA
- Suggerimenti layout (cosa va sopra/sotto la fold)
- P.S. finale
- Note per Systeme.io / GHL (come implementare la pagina)

## Framework di vendita disponibili

### 1. PAS — Problem / Agitate / Solution
**Quando usarlo:** prodotti che risolvono un dolore specifico.
**Perfetto per:** ebook, corsi, tool, servizi.

```
PROBLEM  → Identifica il dolore con precisione chirurgica
AGITATE  → Amplifica le conseguenze del non-agire
SOLUTION → Presenta il prodotto come via d'uscita naturale
PROOF    → Dimostra che funziona (testimonianze, risultati)
OFFER    → Presenta l'offerta con stack di valore
CTA      → Chiama all'azione
```

### 2. AIDA — Attention / Interest / Desire / Action
**Quando usarlo:** prodotti aspirazionali, corsi premium.
**Perfetto per:** corsi avanzati, workshop, consulenza.

```
ATTENTION → Hook potente (headline + sub-headline)
INTEREST  → Perché questo è diverso da tutto il resto
DESIRE    → La trasformazione — come sarà la vita DOPO
ACTION    → L'offerta irresistibile + CTA
```

### 3. STORY-BASED — La storia di Leonardo
**Quando usarlo:** pagine dove la credibilità personale è la leva.
**Perfetto per:** corso Facilmente, brand positioning, primo contatto.

```
LA MIA STORIA → Il viaggio di Leonardo
IL MOMENTO    → La scoperta/svolta che ha cambiato tutto
LA SOLUZIONE  → Quello che ho creato per te
LA PROVA      → Risultati, testimonianze, numeri
L'INVITO      → Unisciti / acquista / partecipa
```

### 4. OTO FRAMEWORK — Per le offerte post-acquisto
**Quando usarlo:** pagine OTO1, OTO2, downsell.
Il cliente ha GIÀ comprato — non serve rivendere la credibilità.

```
CONGRATULAZIONI  → Conferma la scelta appena fatta
IL PASSO SUCC.   → "Ora che hai X, c'è qualcosa che lo completa"
L'OFFERTA SPEC.  → Solo per chi ha appena acquistato
LA SCELTA        → Bottone SÌ grande + link "no grazie" piccolo
```

## Struttura della Sales Letter — Template completo

### ABOVE THE FOLD (quello che si vede senza scrollare)

```
[PRE-HEADLINE — una riga che qualifica il lettore]
"Per geometri e topografi che vogliono [risultato]"

[HEADLINE — la promessa principale, max 15 parole]
La headline deve contenere: il RISULTATO + per CHI + senza OBIEZIONE
Esempio: "Dimezza il tempo delle relazioni tecniche con l'AI 
— anche se non hai mai usato un prompt"

[SUB-HEADLINE — espande la promessa, 1-2 righe]
"Il sistema completo per geometri che vogliono lavorare meglio, non di più"

[VISUAL — hero image o video]
Suggerimento: video di Leonardo che parla (2-3 minuti) o immagine del prodotto

[CTA #1 — il primo bottone, per chi è già convinto]
"VOGLIO LAVORARE MEGLIO →"
```

### IL PROBLEMA (below the fold)

```
[Apri con una situazione riconoscibile]
"Sono le 19. Sei ancora in studio. La relazione tecnica non è finita.
Il cliente chiama per la terza volta. I tuoi figli cenano senza di te."

[Amplifica — 3-4 paragrafi brevi che scavano nel dolore]
Dialoghi reali:
- "Questi file sono inutilizzabili!"
- "Mi aspettavo un progetto più veloce. Cos'è che sta rallentando?"
- "Perché devi fare sempre tutto all'ultimo minuto?"

[Il costo del non-agire]
"Ogni settimana persa sono [X ore / Y euro] che non recuperi."
```

### LA SVOLTA

```
[Transizione — dal problema alla speranza]
"E se ci fosse un modo per [risultato principale]?
Non un trucchetto. Un sistema completo."

[Presentazione di Leonardo — breve, credibile]
"Mi chiamo Leonardo Massafra. Faccio il geometra da 20 anni.
Ho rilevato il Colosseo, l'Amerigo Vespucci, siti in mezzo mondo.
E due anni fa ho scoperto qualcosa che ha cambiato il mio modo di lavorare."
```

### LA SOLUZIONE (il prodotto)

```
[Nome del prodotto — con positioning]
"Ecco [NOME PRODOTTO]: il [tipo] pensato per [chi]"

[Stack di valore con bullet points]
✅ [Componente 1] — [beneficio in una riga] (Valore: €XX)
✅ [Componente 2] — [beneficio in una riga] (Valore: €XX)
✅ [Componente 3] — [beneficio in una riga] (Valore: €XX)
✅ BONUS: [Bonus] — [beneficio] (Valore: €XX)

📦 Valore totale: €[somma]
💰 Oggi tuo a soli: €[prezzo]
```

### LA PROVA

```
[Testimonianze — le più forti e specifiche]
- Dott. Leonardi (Biblioteca Capitolare)
- Ing. Piovan
- Arch. Sinicato
- Arch. Fasoli

[Risultati numerici]
"225 geometri hanno già [usato/partecipato/scaricato]"
"Il 73% ha [risultato specifico]"

[Credenziali]
"Formatore scelto dal Consiglio Nazionale Geometri"
"Autore del libro 'Misure Perfette'"
"Rilievi in 4 continenti"
```

### L'OFFERTA

```
[Ricapitolo chiaro e visivo]
"Ricapitolando, con [PRODOTTO] ricevi:
→ [Componente 1]
→ [Componente 2]
→ [Componente 3]
→ [Bonus]
Il tutto a soli €[prezzo]"

[GARANZIA]
"Garanzia soddisfatti o rimborsati entro [X] giorni.
Se non sei soddisfatto, ti rimborso senza fare domande.
Il rischio è tutto mio."

[CTA #2 — il bottone principale]
"VOGLIO LAVORARE MEGLIO" / "SÌ, VOGLIO IL KIT" / "ACCEDO ORA"
MAI: "Acquista" / "Compra" / "Paga" (troppo transazionale)
```

### FAQ

```
[3-5 domande che risolvono le obiezioni principali]
- "Potrei fare da solo" → [risposta]
- "Servizi simili mi hanno già deluso" → [risposta]
- "Non sono sicuro che valga il costo" → [risposta]
- "Temo che ci vorrà più tempo del promesso" → [risposta]
```

### CHIUSURA + P.S.

```
[Ultimo appello — emotivo, breve]
"La scelta è tua. Puoi continuare a [problema] oppure puoi 
[risultato]. Io ho fatto la mia scelta due anni fa. 
Non sono mai tornato indietro."

[CTA #3 — ultimo bottone]

[P.S. — spesso la parte più letta della pagina]
"P.S. Ogni giorno senza [soluzione] è un giorno dove 
[conseguenza concreta del problema]."
```

## Template specifici per il funnel

### Pagina OTO1 (post-acquisto, 47-67€)

```
HEADLINE: "Aspetta — prima di andare, ho qualcosa che completa 
perfettamente quello che hai appena preso."

CORPO (max 500 parole):
"Hai appena [acquistato X]. Ottima scelta.

Ma c'è una cosa che [X] non copre: [gap specifico].

Per questo ho creato [OTO1]: [descrizione breve].

Dentro trovi:
→ [Componente 1]
→ [Componente 2]
→ [Componente 3]

Normalmente costa €[prezzo pieno]. 
Ma siccome hai appena acquistato, te lo offro a soli €[prezzo scontato].

Questa offerta è disponibile solo adesso."

CTA: "SÌ, AGGIUNGI AL MIO ORDINE — €[prezzo]"
Link sotto: "No grazie, passo al prossimo step →"
```

### Pagina OTO2 (seconda offerta, 97-147€)

```
HEADLINE: "Un'ultima cosa — il pacchetto completo"

CORPO (max 400 parole):
Stessa struttura OTO1 ma offerta di livello superiore.
Enfasi sul "tutto incluso" e sul risparmio rispetto all'acquisto singolo.

CTA: "VOGLIO IL PACCHETTO COMPLETO — €[prezzo]"
Link sotto: "No grazie, prosegui →"
```

### Pagina Downsell (per chi rifiuta l'OTO)

```
HEADLINE: "Capisco — forse [OTO] era troppo in questo momento. 
Che ne dici di questo?"

CORPO (max 300 parole):
Offerta ridotta — meno componenti, prezzo inferiore.
"Solo [componente più importante] a €[prezzo ridotto]"

CTA: "SÌ, QUESTO MI INTERESSA — €[prezzo]"
Link sotto: "No grazie, concludi l'ordine →"
```

## Headline — Banca di formule

**Risultato + tempo:**
> "Come [risultato] in [tempo] — anche se [obiezione]"

**Domanda che qualifica:**
> "Sei un geometra che [problema]? Ecco la soluzione."

**Numero + beneficio:**
> "[X] [cose/errori/segreti] per [risultato]"

**Provocazione + curiosità:**
> "Perché [cosa controintuitiva] (e come ti aiuta)"

**Testimonianza come headline:**
> "'Ha cambiato il mio modo di lavorare' — [Nome, professione]"

## Testi bottone CTA — Banca

**Orientati al RISULTATO:**
- "VOGLIO LAVORARE MEGLIO"
- "SÌ, VOGLIO DIMEZZARE I TEMPI"
- "ACCEDO AL SISTEMA COMPLETO"
- "SCARICA IL KIT ADESSO"
- "PRENOTA IL MIO POSTO"
- "INIZIA ORA — €[prezzo]"

**Mai usare:** "Compra" / "Acquista" / "Paga" / "Invia" / "Clicca qui"

## Adattamento per segmento

### Per architetti/ingegneri (servizi di rilievo)
- **Enfasi:** precisione, affidabilità, file utilizzabili, risparmio tempo
- **Linguaggio:** tecnico-professionale, dati concreti
- **Prove:** Colosseo, Biblioteca Capitolare, Coca-Cola HBC
- **Obiezioni:** "già deluso da altri", "costo extra", "file pesanti"

### Per geometri corsisti (corsi AI)
- **Enfasi:** risparmio tempo, competitività, semplicità
- **Linguaggio:** colloquiale, pratico, "come al corso"
- **Prove:** feedback dai corsi, Consiglio Nazionale, 225 risposte quiz
- **Obiezioni:** "non sono tecnologico", "costa troppo", "non ho tempo"

### Per topografi (ebook GPS)
- **Enfasi:** errori da evitare, precisione, esperienza sul campo
- **Linguaggio:** tecnico ma accessibile, da collega
- **Prove:** 90+ vendite, 20 anni esperienza, progetti internazionali
- **Obiezioni:** "so già tutto", "è solo teoria", "trovo gratis online"

## Principi che la skill applica SEMPRE

1. **Una pagina = un prodotto = un'azione** — mai confondere
2. **Vendi la trasformazione, non il prodotto** — non "30 template" ma "relazioni in metà tempo"
3. **Specifica sempre i numeri** — "27 template" non "molti template"
4. **L'obiezione non affrontata è una vendita persa** — rispondi a TUTTE nelle FAQ
5. **La garanzia riduce il rischio percepito** — inseriscila sempre
6. **Il P.S. è la seconda cosa più letta dopo la headline** — usalo
7. **CTA visibile sempre** — almeno 3 bottoni (sopra, metà, fondo)
8. **Social proof vicino al CTA** — testimonianza breve sopra il bottone
9. **Urgenza solo se reale** — scarsità finta distrugge la fiducia
10. **Mobile first** — 70%+ legge da smartphone, frasi corte, paragrafi brevi

## Workflow

1. L'utente fornisce: tipo pagina, prodotto, target, posizione nel funnel
2. Scelgo il framework più adatto (PAS / AIDA / Story / OTO)
3. Identifico le obiezioni principali del segmento target
4. Scrivo il copy completo sezione per sezione
5. Fornisco 2 varianti A/B per la headline
6. Aggiungo suggerimenti per Systeme.io / GHL (layout, colori, posizione elementi)
